What is it?

Essense is IDA Pro (Russian) plugin. It's main purpose is to save essense of IDA database to XML-storage. It's like Dump database to IDC IDA command, but with some difference: essense is function-based (by default, plugin will save only user-modified functions). Next, plugin is like IDA signatures - try to save only 'valuable' instruction length, so essense is become position and file independent.
This can be useful, if you analysed some version of file and now try to load newer version. Or you analyse several files, which use same library/API. You can just save selected function from current database to XML-file (essense) and then just find and load them in new database.You will load almost everything:
  • Structures used in this function
  • Enums used in this function
  • Frame of function
  • Names, comments, etc...
Also, you will see a lot of bugs, of coz! :-)

Usage

You can load the plugin by pressing (by default) "Alt-1". Choose requred action (Save or Load). Plugin options:
  • Start Address - start address for operation
  • End Address - end address for operation
  • Filename - file name for operation
  • Show debug info - show some unuseful info :-)
  • Ignore CALL TO
    To recognize function like 
    func_x:
    call func_y
    ret
    plugin have to store name of func_y and check it while loading. To turn off this option, check this flag. But be ready to edit/delete some small functions from XML-file or you have a lot of wrong-recognized functions in new database.
    On other side - sometimes you have to run plugin several times to have function func_y recognized, before function func_x will be recognized.
  • Don't check names of functions - save all functions, not just user-named.
    Be careful - this will grow a lot you XML file!
  • All functions - save all functions, detected by IDA.
    Don't check previous flag to save all user-named functions in database
  • Don's save/restore structures - do not save any extra info. Recommended for standard complex structures, like IRP or something like.
  • Force load - by default, if function already have user-name, it will be skipped at load stage. To force loading - set this flag.
